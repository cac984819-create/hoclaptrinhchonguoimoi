JavaScript Deep Internals

Khám phá cơ chế Memory Heap, Call Stack và thế giới lập trình bất đồng bộ.

1. JS Engine & Runtime Environment

JavaScript là Single-threaded (đơn luồng), nghĩa là nó chỉ có thể làm một việc tại một thời điểm. Vậy làm sao nó xử lý hàng ngàn request cùng lúc?

  • Call Stack: Nơi theo dõi các hàm đang được thực thi (LIFO - Last In, First Out).
  • Memory Heap: Nơi cấp phát và lưu trữ bộ nhớ cho biến và đối tượng.
  • Event Loop: "Cảnh sát giao thông" điều phối các tác vụ từ Callback Queue vào Stack khi Stack trống.

2. Scope, Hoisting & Closures

Đây là những khái niệm "hack não" nhất nhưng lại quan trọng nhất trong JS chuyên nghiệp.

Closure là gì?

Là khả năng một hàm ghi nhớ và truy cập vào phạm vi (scope) của nó ngay cả khi hàm đó đã thực thi xong.

function outer() { const message = "Chào bạn!"; return function inner() { console.log(message); // Ghi nhớ biến message } }

3. Lập trình bất đồng bộ (Async/Await)

Để tránh việc trang web bị "đơ" khi tải dữ liệu, chúng ta sử dụng cơ chế bất đồng bộ.

  • Promises: Lời hứa cho một giá trị trong tương lai (Pending, Fulfilled, Rejected).
  • Async/Await: Cú pháp hiện đại giúp viết code bất đồng bộ trông như đồng bộ, dễ đọc và bảo trì hơn.
Microtasks vs Macrotasks: Promises có độ ưu tiên cao hơn setTimeout. Event Loop luôn ưu tiên dọn sạch Microtask queue trước.

4. ES6+ & Clean Code Patterns

Viết code ngắn gọn và hiệu quả với các tính năng mới nhất của ECMAScript.

// Destructuring & Spread Operator const user = { name: 'Gemini', age: 1 }; const { name } = user; const newUser = { ...user, location: 'Cloud' }; // Arrow Functions & Template Literals const greet = (n) => `Hello, ${n}!`;

Module System: Sử dụng import và export để chia nhỏ dự án thành các phần dễ quản lý.
